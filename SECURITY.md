# Security Policy

## サポート対象

個人開発のため、`main` ブランチの最新版のみをサポート対象とします。過去バージョンへのセキュリティパッチのバックポートは行いません。

## 脆弱性の報告

セキュリティ上の問題を発見した場合は、**公開のGitHub Issueを立てないでください**。第三者に悪用される前に非公開で報告してください。

- 推奨: このリポジトリの [Security Advisories](../../security/advisories/new) から非公開レポートを作成
- または: sanbongazin@gmail.com へメールで連絡

報告には可能な範囲で以下を含めてください。

- 再現手順
- 影響範囲(該当する権限・データ・ユーザー)
- 想定される悪用シナリオ

## 対応方針

- 報告受領後、可能な限り速やかに一次返信します(個人開発のため即日対応を保証するものではありません)
- 修正版がリリースされるまで、詳細の公開は控えていただくようお願いします
- 本拡張機能はBOOTHの非公式ツールであり、パスワード等の認証情報を外部送信することはありません。LLM API連携機能はユーザー自身のAPIキーを使用し、拡張機能側で秘密鍵を保持・送信しません

## サプライチェーンセキュリティ

- 依存パッケージは Dependabot により週次で脆弱性・更新をチェックしています
- CIでは `npm ci`(lockfile厳密一致)でのインストールと `npm audit` を実行しています
- GitHub Actions は tag ではなくコミットSHAでピン留めしています
